A Model for Predicting Fraudulent Loan Applications Using Machine Learning

Best model: XGBoost

Supervised by Bernard Shibwabo. Financial institutions lose ground to loan fraud during the verification stage, where forged documents and fabricated identities are hardest to catch — and traditional, manual review can't keep pace. This study builds and compares three supervised models to catch fraudulent applications earlier and more reliably.

Approach

  • Sourced a 50,001-record structured loan application dataset (Kaggle) with income, employment, credit score, loan amount, repayment history, and fraud labels.
  • Built a pre-processing pipeline: missing-value checks, label encoding for categorical fields, and feature scaling with StandardScaler.
  • Corrected severe class imbalance with SMOTE, balancing fraud and legitimate classes at 39,161 records each before an 80/20 stratified train-test split.
  • Trained and compared Logistic Regression (interpretable baseline), Random Forest, and XGBoost (n_estimators=100, learning rate 0.1, max depth 6), tuned via GridSearchCV.

Why it matters

XGBoost flagged nearly every fraudulent application in the test set — a deliberate trade-off, since missing a fraud case is far costlier to a lender than sending a legitimate one for manual review. The findings support folding gradient-boosted ensembles into automated loan screening as a scalable complement to human review, with future work extending to institution-specific data and document-level fraud signals.
